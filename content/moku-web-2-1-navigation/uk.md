Є вірна ознака, що фреймворк тобі бреше: твій застосунок обростає обхідними шляхами, і в кожного обхідного шляху довгий, винуватий коментар, який пояснює, чому він узагалі мусить існувати. Не «дивись, як хитро», а «це тут, бо те, що під ним, не дає мені зробити очевидну річ».

Я впʼявся в цю стіну, коли збирав **Atlas** — редизайн демо Tracker, real-time канбан-дошку на `@moku-labs/web`. Воно працює. Воно гарне. І кодова база тихенько обзавелася маленьким музеєм виправдань:

- Хелпер `navigate()`, який **створював відʼєднаний елемент `<a>`, клікав по ньому й видаляв** — бо програмна навігація не мала іншого способу змусити SPA реально перемкнутися.
- Цілий модуль, `board-scroll.ts`, що пришпилював `<body>` через `position: fixed; top: -scrollY`, щоб дошка не сіпалася на самий верх, коли ти відкривав над нею issue.
- Панель issue, що тримала свій *застарілий* вміст у памʼяті й перемикала атрибут `hidden`, бо повернення порожнього рендера ламало дерево Preact, і панель уже ніколи не відкривалася знову. У коментарі так і було написано: *«відкривається лише найперший issue»*.
- 270-рядковий `realtime.ts`, що вручну ліпив reconnect WebSocket, keepalive і буфер для пре-сіду.

Чотири хаки. Чотири діри. Чесний хід — не писати пʼятий обхідний шлях, а спуститися на рівень нижче й зробити очевидну річ можливою. Тож я випустив **moku-web 2.1**: пʼять додаткових можливостей, без жодних ламких змін, усі дефолти недоторкані. Ось що це за можливості — і який хак кожна з них списує в архів.

## 1. Острови тепер уміють навігувати

`app.spa.navigate()` існував, але острів до нього не дотягувався — острови не отримують хендл `app`. Тож Atlas синтезував клік по одноразовому якорю, «єдиному шляху, який поважають обидва перехоплювачі». Працювало це так, як працює скотч.

Тепер `navigate` — завжди присутній член контексту острова, поряд із `set`, `cleanup` і `url`:

```ts
events: {
  "click [data-close]": (ctx) => ctx.navigate(ctx.url("board", { id: ctx.params.id })),
}
```

Жодного `app`, жодного якоря, жодного сміття в DOM.

## 2. Вигляд — це властивість екрана

Atlas хотів, щоб перехід дошка↔дошка відчувався як одне, а відкриття issue — як інше. Фреймворк давав йому один глобальний булеан: view transitions ввімкнено або вимкнено. Один кросфейд на все.

Тепер переходи оголошуються на маршруті — типізована директива, а **не** довільний мішок `.meta()` (фреймворк її інтерпретує, тож вона заслуговує на справжній метод зі справжнім автокомплітом):

```ts
route("/board/{id}").transition("slide")
route("/board/{id}/issue/{issueId}").transition("morph")
```

Ядро тегує кожен перехід, щоб твій CSS міг його стилізувати (`:root[data-view-transition~="slide"]` або стандартний `:active-view-transition-type()`), а спільний `view-transition-name`, лишений по обидва боки свопу, переллє один елемент в інший. `spa.viewTransitions` стає дефолтом на весь застосунок; маршрут його перевизначає.

## 3. Скрол — це директива, а не хак

`board-scroll.ts` — заморозка через `position: fixed` — існував рівно з однієї причини: SPA завжди скролив на самий верх під час навігації, а маршрут-оверлей потребує, щоб сторінка *лишилася на місці*. Увесь файл був застосунком, який нейтралізує фреймворк.

Ядро вже мало всередині опцію відмови від скролу на конкретну навігацію. Просто до неї було не дотягнутися. Тепер можна:

```ts
route("/board/{id}/issue/{issueId}").scroll("preserve")   // open the overlay; don't move the board
```

Плюс дефолт `spa.scrollRestoration` на весь застосунок і `ctx.navigate(path, { scroll: "preserve" })` на конкретний виклик. Atlas видалив `board-scroll.ts` і весь його танець lock/unlock повністю. Одне слово в метаданих маршруту замінило цілий файл.

## 4. «Нічого не рендерити», що насправді працює

Це був баг, перевдягнений в обхідний шлях. Персистентний острів, який повертав порожній рядок, виконував `host.innerHTML = ""`, що видаляє DOM з-під утримуваного віртуального дерева Preact — тож *наступному* рендеру не було проти чого діфати, і він мовчки нічого не робив. Звідси «відкривається лише найперший issue», звідси й застосунок, що вічно тримав застарілий стан, аби тільки ніколи не рендерити порожнечу.

Тепер рендер може повернути `null` — нічого не рендерити, але лишатися монтованим. Це йде через власний `render(null, host)` Preact, чистий анмаунт, який потім перекомічується:

```ts
render: (s) => (s.open ? h(IssuePanel, { issue: s.issue }) : null),
```

Панель issue видалила всю свою бухгалтерію з утримання застарілого стану і просто… закривається.

## 5. WebSocket, який няньчить сам себе

Кожен real-time застосунок переписує той самий менеджер сокета: reconnect з backoff, keepalive-пінг, буфер для фреймів, що приходять раніше за твій снапшот, фан-аут на хендлери. `createChannel` — це воно, один раз, у фреймворку:

```ts
const board = createChannel<BoardPatch>({
  url: (id) => `${location.origin.replace(/^http/, "ws")}/ws/board/${id}`,
  keepAlive: { send: "ping", ignore: "pong" },
  bufferUntilSeed: true,
});
const off = board.subscribe(boardId, (patch) => applyPatch(ctx, patch));
```

`subscribe` рахує посилання — перший підключається, останній відключається — тож два острови, що ділять одну дошку, перестають бути несучою конструкцією, залежною від порядку маунту. `realtime.ts` з Atlas усох із 270 рядків водопроводу до тонкого адаптера. Воно існує лише в браузері й виструшується tree-shaking-ом, якщо ти ніколи не відкриваєш канал.

## Той момент, де тести збрехали

Ось шматок, який я лишаю, бо це і є справжній урок. Усі пʼять фіч мали юніт-тести. 950 штук, зелені. Я випустив 2.1.0.

А потім я зробив те, чого тести зробити не можуть: відкрив справжній застосунок у справжньому браузері й клікнув по картці. Панель відкрилася. Скрол лишився на місці. А в адресному рядку досі стояло `/`.

`app.spa.navigate` міняв контент і оновлював свою внутрішню уяву про URL — але ніколи не викликав `history.pushState`. Клік по лінку проходить крізь навігаційну машинерію браузера, яка коммітить URL за тебе; а *програмний* виклик усе це оминає. Тож сторінка змінилася, а адресний рядок, кнопка «назад», рефреш і кожен deep link лишилися на старому URL. Мій новий `ctx.navigate` успадкував цю діру. Тести бігали в DOM, у якого немає справжньої history, тож вони були ідеально, упевнено зеленими.

Оце і є `2.1.1`: ядро саме коммітить URL ще до свопу. Спіймано ціною одного кліку по картці. Зелені тести — це не робочий застосунок; вони ним ніколи й не були.

## Про що це насправді

Жодна з цих пʼяти речей не є великою. У тому й суть. Фреймворк заробляє довіру в дрібних моментах — коли очевидна річ є водночас і доступною річчю, і твій застосунок перестає потребувати музей виправдань. Кожна можливість тут прийшла прямо зі списку хаків справжнього демо, і її випуск видалив той хак. Це єдиний feature request, якому я довіряю: *обхідний шлях, який моєму власному застосунку довелося написати*.

`@moku-labs/web@2.1.1` уже на npm. Демо Atlas змігровано. На чотири хаки легше.
