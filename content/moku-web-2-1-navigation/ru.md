Есть верный признак того, что фреймворк тебе врёт: приложение обрастает обходными путями, и у каждого обходного пути — длинный извиняющийся комментарий, объясняющий, почему он вообще тут. Не «смотри, как хитро придумано», а «это здесь, потому что то, что под капотом, не даёт мне сделать очевидное».

Я упёрся в эту стену, когда собирал **Atlas** — редизайн демки Tracker, real-time канбан-доску на `@moku-labs/web`. Она работает. Она приятная. И в кодовой базе тихо вырос целый музей извинений:

- Хелпер `navigate()`, который **создавал отдельный элемент `<a>`, кликал по нему и удалял** — потому что у программной навигации не было другого способа заставить SPA реально переключиться.
- Целый модуль `board-scroll.ts`, который прибивал `<body>` через `position: fixed; top: -scrollY`, чтобы доска не отскакивала наверх, когда ты открываешь поверх неё issue.
- Панель issue, которая держала своё *устаревшее* содержимое в памяти и переключала атрибут `hidden`, потому что возврат пустого рендера ломал дерево Preact, и панель уже никогда не открывалась заново. В комментарии буквально было написано: *«открывается только самый первый issue»*.
- 270-строчный `realtime.ts`, вручную реализующий переподключение WebSocket, keepalive и буфер предзагрузки.

Четыре костыля. Четыре дыры. Честный ход — не писать пятый обходной путь, а спуститься на уровень ниже и сделать очевидное возможным. Так что я выкатил **moku-web 2.1**: пять аддитивных возможностей, без ломающих изменений, ни один дефолт не тронут. Вот что каждая из них собой представляет и какой костыль отправляет на пенсию.

## 1. Острова теперь умеют навигировать

`app.spa.navigate()` существовал, но остров не мог до него дотянуться — островам не выдают ручку `app`. Так что Atlas синтезировал клик по одноразовому якорю, «единственному пути, который чтят оба перехватчика». Работало это так же, как работает изолента.

Теперь `navigate` — всегда присутствующий член контекста острова, прямо рядом с `set`, `cleanup` и `url`:

```ts
events: {
  "click [data-close]": (ctx) => ctx.navigate(ctx.url("board", { id: ctx.params.id })),
}
```

Никакого `app`, никакого якоря, никакого мусора в DOM.

## 2. Внешний вид — это свойство экрана

Atlas хотел, чтобы переход board↔board ощущался как одно, а открытие issue — как другое. Фреймворк давал ему один глобальный булеан: view-переходы включены или выключены. Один кроссфейд на всё.

Переходы теперь объявляются на маршруте — типизированная директива, **а не** свободный мешок `.meta()` (фреймворк интерпретирует её, так что она заслуживает настоящего метода с настоящим автокомплитом):

```ts
route("/board/{id}").transition("slide")
route("/board/{id}/issue/{issueId}").transition("morph")
```

Ядро помечает каждый переход, чтобы твой CSS мог его стилизовать (`:root[data-view-transition~="slide"]` или стандартизируемый `:active-view-transition-type()`), а общий `view-transition-name`, оставшийся между свопами, превратит один элемент в другой. `spa.viewTransitions` становится дефолтом на уровне приложения; маршрут его переопределяет.

## 3. Скролл — это директива, а не костыль

`board-scroll.ts` — заморозка через `position: fixed` — существовал ровно по одной причине: SPA всегда скроллил наверх при навигации, а маршруту-оверлею нужно, чтобы страница *оставалась на месте*. Весь файл был приложением, нейтрализующим фреймворк.

В ядре уже был внутренний пер-навигационный отказ от скролла. Просто до него было не дотянуться. Теперь — можно:

```ts
route("/board/{id}/issue/{issueId}").scroll("preserve")   // открой оверлей; не двигай доску
```

Плюс дефолт `spa.scrollRestoration` на уровне приложения и пер-вызов `ctx.navigate(path, { scroll: "preserve" })`. Atlas целиком удалил `board-scroll.ts` и его танцы с lock/unlock. Одно слово метаданных маршрута заменило целый файл.

## 4. «Не рендерить ничего», которое реально работает

Это был баг в костюме обходного пути. Персистентный остров, возвращавший пустую строку, выполнял `host.innerHTML = ""`, что вырезает DOM прямо из-под удержанного виртуального дерева Preact — и *следующему* рендеру было нечего диффить, и он молча превращался в no-op. Отсюда «открывается только самый первый issue» и отсюда же приложение, вечно хранящее устаревшее состояние, лишь бы никогда не рендерить пустоту.

Рендер теперь может вернуть `null` — не рендерить ничего, но остаться примонтированным. Это проходит через собственный `render(null, host)` Preact — чистый анмаунт, который позже коммитится заново:

```ts
render: (s) => (s.open ? h(IssuePanel, { issue: s.issue }) : null),
```

Панель issue удалила всю свою бухгалтерию по хранению устаревшего состояния и просто... закрывается.

## 5. WebSocket, который сам за собой следит

Каждое real-time приложение заново переписывает один и тот же менеджер сокета: переподключение с бэкоффом, keepalive-пинг, буфер для фреймов, прилетевших до загрузки твоего снимка, раздача обработчикам. `createChannel` — это всё, один раз, во фреймворке:

```ts
const board = createChannel<BoardPatch>({
  url: (id) => `${location.origin.replace(/^http/, "ws")}/ws/board/${id}`,
  keepAlive: { send: "ping", ignore: "pong" },
  bufferUntilSeed: true,
});
const off = board.subscribe(boardId, (patch) => applyPatch(ctx, patch));
```

`subscribe` считает ссылки — первый вошедший подключается, последний вышедший отключается — так что два острова, делящие одну доску, перестают зависеть от порядка монтирования. `realtime.ts` в Atlas усох с 270 строк проводки до тонкого адаптера. Он только для браузера и вытряхивается tree-shaking-ом, если ты никогда не открываешь канал.

## Часть, где тесты соврали

Вот кусок, который я оставляю, потому что в нём и есть настоящий урок. У всех пяти фич были юнит-тесты. 950 штук, все зелёные. Я выкатил 2.1.0.

А потом я сделал то, чего тесты сделать не могут: открыл настоящее приложение в настоящем браузере и кликнул по карточке. Панель открылась. Скролл остался на месте. А в адресной строке всё ещё было `/`.

`app.spa.navigate` подменил контент и обновил своё внутреннее представление об URL — но так и не вызвал `history.pushState`. Клик по ссылке проходит через навигационную машинерию браузера, которая коммитит URL за тебя; а *программный* вызов всё это обходит. Так что страница менялась, пока адресная строка, кнопка «назад», обновление и каждый deep link оставались на старом URL. Мой новый `ctx.navigate` унаследовал эту дыру. Тесты гоняли в DOM, у которого нет настоящего history, так что они были безупречно, уверенно зелёными.

Это и есть `2.1.1`: ядро коммитит URL само, до свопа. Поймано ценой одного клика по карточке. Проходящие тесты — это не работающее приложение; они им никогда и не были.

## О чём это на самом деле

Ни одна из этих пяти штук не большая. В этом и суть. Фреймворк зарабатывает доверие в мелочах — когда очевидное оказывается ещё и доступным, и твоё приложение перестаёт нуждаться в музее извинений. Каждая возможность здесь пришла прямиком из списка костылей настоящей демки, и её выкатывание удалило костыль. Это единственный feature request, которому я доверяю: *обходной путь, который пришлось написать моему собственному приложению.*

`@moku-labs/web@2.1.1` уже в npm. Демка Atlas мигрирована. На четыре костыля легче.
