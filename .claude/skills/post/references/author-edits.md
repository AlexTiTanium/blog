# Author's learned preferences (auto-maintained)

This file accumulates the **recurring patterns** of the author's review-loop edits, so each new
post starts pre-calibrated instead of repeating the same corrections.

**Read** at SKILL.md Step 2 (before drafting) and apply every entry from the first draft.
**Append** at SKILL.md Step 8.6 (when the review loop ends) — but only patterns, never one-off
edits:

- A pattern is a *kind* of change that plausibly generalizes to future posts ("cuts explanatory
  closers", "prefers «ты» register in Russian", "never wants emoji").
- A one-off is content-specific ("renamed the function in the example") — don't log it.
- If a new pattern contradicts an existing entry, **update the entry** (the author changed their
  mind); don't keep both.
- Keep entries short — one line each, newest last. If this list grows past ~20 entries,
  consolidate duplicates.

## Format

```
- YYYY-MM-DD <slug>: <the pattern, one line>
```
